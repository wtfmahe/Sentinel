# Welcome

Thank you for contributing to Sentinel.

Our goal is to build the best open-source autonomous AI platform.

Every contribution should improve the platform while keeping the architecture clean, modular, and maintainable.

# Development Principles

* Keep components modular.
* Prefer composition over duplication.
* Document architectural decisions.
* Write clean, readable code.
* Follow existing conventions.

---

# Branch Naming

Examples:

feature/job-agent

feature/memory-engine

fix/api-authentication

docs/update-roadmap

---

# Commit Format

Use Conventional Commits.

Examples:

feat(agent): add browser agent

feat(memory): implement retrieval service

fix(api): validate authentication

docs: update roadmap

refactor(worker): simplify queue handling

---

# Pull Requests

Every pull request should include:

* Clear description
* Reason for the change
* Testing performed
* Documentation updates if required

---

# Reporting Issues

Please include:

* Expected behavior
* Actual behavior
* Steps to reproduce
* Environment
* Logs (if available)

---

# Code Reviews

Reviews should focus on:

* Correctness
* Readability
* Maintainability
* Performance
* Security

---

Every contribution helps make Sentinel better.
